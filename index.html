    Embedded Game Console

    Flappy Bird Demo
    Links
    GitHub Link: https://github.com/relientm96/gameconsole

    About

    Project involving the design and development of a game console electronic embedded system made from scratch. This was a guided class project but personally added various extensions to the software program of the micro-controller including a battery life reader, brightness control, C library that renders characters onto the LCD screen and mulitple classic arcade games.

    Personal Games Programmed

    • Flappy Bird (simplified)
    • Etch A Sketch
    • Pong (with two level AIs)
    • Monty Hall

    Features of Device

    • Boost Converter power circuit to step up from 1.5V battery source to 3.3 V.
    • Screen brightness control by adjusting Pulse Width Modulation (PWM) frequency.
    • AVR Studio as an IDE to flash C code onto micro-controller via a JTAG board.
    • C character library used for LCD character displays.
    • Micro-controller interrupt used for event driven program including timer and button interrupts.
    • Battery level detector using Analog to Digital Conversion.
    • Serial Peripheral Interface(SPI) and bit banging communication between micro-controller to LCD.

    Project Development Chronology

    • Planned and implemented circuit functionality onto schematics on Altium Designer.
    • Routed and placed components to design Printed Circuit Board.
    • Soldered on surface mount components.
    • Programmed simple features on micro-controller using C, JTAG and AVR Studio.
    • Added various self made games and features to game console device.

    Tweet Analysis Cloud Application

    Links
    GitHub Link: https://github.com/relientm96/SinsOnTwitter

    About

    Worked in a group of five to develop a cloud based cluster application that analyzes how "sinful" Australians are on twitter based on the Seven Deadly Sins . Tweets are harvested, analysed, processed and stored onto a cluster database and results of analysis are displayed to users onto an interactive front end web application. Users could search keywords and select a particular sin to observe the amount of tweets with that word throughout each state through interactive charts. My main role in the team was to integrate system parts (Web application, database and tweet harvester) into a single three machine cluster system and deploy it onto the NecTAR cloud.

    Application Features

    • Uses a NoSQL CouchDB three node database cluster system.
    • Tweets harvested across Australia through Python's Tweepy Library.
    • Around 600,000 tweeets stored across three 80 GB volumes on the cloud.
    • Tweets processed and analysed using sentiment analysis and word indexing.
    • Web application designed using Flask and interactive charts created using Chart.js.
    • Application hosted on three Ubuntu Linux based virtual machines.

    Tools used in project

    • Ansible for automated system deployment which includes cloud resource provisioning and Linux bash commands.
    • CouchDB three node cluster setup using Docker for containerization.
    • Linux operating system command tools and environment (Ubuntu 18.04).
    • Apache2 as the HTTP server which proxies requests to the web application through mod_wsgi

    Online C Code Editor


    Links
    Application Link: http://13.229.230.199/
    GitHub Link: https://github.com/relientm96/online_code_editor

    About

    Made a Full Stack online C Code integrated development environment web application where users can log on to run C programs. Backend server created using C++ and POCO C++ Libraries and deployed onto an EC2 Windows server instance on the AWS cloud. Simple code samples are also provided to users.

    How it works

    When a user creates a project from the login screen, a .c file is created on the server's file system. Once the user is happy to compile their program, they would click 'compile' which sends the user's code on the editor as a text payload to the server. The text is copied into the .c file and the GCC compiler compiles the file. The server then responds by sending either the error output if there is an error in compilation or the successful compiled file output to the browser. Once the user closes the site, an automatic "DELETE" request is made to the server to delete the user's .c and .exe files to clear space.

    Features

    • C++ Backend with GCC 6.3.0 C compiler
    • FrontEnd user code text editor interface
    • Sample simple C programs provided
    • Registration feature for users

    Utilized

    • POCO C++ Libraries
    • CodeMirror as code editor
    • HTML, CSS, Javascript and Bootstrap for front end web design
    • RESTAPI calls to backend and handling of JSON payload.

    Seven Segment on Web

    Links
    GitHub Link: https://github.com/relientm96/SevenSegment

    About

    An arduino based project that allows for Seven Segment Digit control over a Web interface.
    The ESP8266 is used as a Web Server that sends out a html webpage for user interaction.
    The Arduino UNO is used to collect received serial data (transmitted through UART) from the ESP8266 and displays the number onto the double digit seven segment display.

    Available Modes

    • Potentiometer Mode: Control number to display by reading analog pin connected to Potentiometer.
    • Auto Counter: Arduino automatically increments from 0-99 and displays it.
    • Web Server Control: Utilizes the input number on Web Interface which it sent to the ESP8266 followed by the UNO for display.
